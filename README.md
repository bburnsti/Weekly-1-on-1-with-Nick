# Weekly 1-on-1 with Nick

A dedicated repository for tracking meeting notes, action items, and discussion topics from weekly one-on-one sessions with my supervisor, Nick.

---

## Purpose

This repository serves as a structured record of our ongoing weekly check-ins. It is designed to:

- **Maintain continuity** – Keep a running history of discussions, decisions, and follow-ups across meetings.
- **Track accountability** – Document action items and owners so nothing falls through the cracks.
- **Foster alignment** – Ensure both parties are on the same page regarding priorities, expectations, and progress.
- **Support professional growth** – Capture feedback, goals, and development conversations over time.

---

## Meeting Structure

Each weekly 1-on-1 generally follows this agenda:

1. **Updates & Check-in** – Brief status on current projects and workload.
2. **Action Item Review** – Follow-up on tasks assigned in previous meetings.
3. **Discussion Topics** – Items raised by either party (priorities, blockers, feedback, ideas).
4. **Goals & Development** – Ongoing conversations around career growth and professional development.
5. **New Action Items** – Capture next steps with clear owners and due dates.

---

## Repository Structure

```
/
├── README.md          # This file
└── meetings/          # Folder containing individual meeting notes
    └── YYYY-MM-DD.md  # Meeting notes named by date
```

---

## Meeting Notes Format

Each meeting file follows this template:

```markdown
# 1-on-1 – YYYY-MM-DD

## Action Item Review
- [ ] Item from last week (Owner)

## Discussion Topics
- Topic 1
- Topic 2

## New Action Items
- [ ] New task (Owner) – Due: YYYY-MM-DD

## Notes
Additional context or decisions made during the meeting.
```

---

## Guidelines

- Meeting notes should be added within 24 hours of each session.
- Action items should include a clear owner and target completion date.
- All discussion topics should be added to the relevant meeting file prior to the meeting when possible.
